1. Del-Laundry Management System

Aplikasi manajemen operasional laundry ini dirancang untuk mempermudah pencatatan pesanan, perhitungan biaya, dan pengelolaan data layanan pelanggan secara efisien. Proyek ini sangat berfokus pada struktur kode yang bersih dan implementasi arsitektur berbasis objek.

Sorotan Teknis (Technical Highlights):
  • Object-Oriented Programming (OOP): Implementasi pilar OOP terlihat jelas pada struktur sistem. Encapsulation digunakan untuk melindungi akses langsung ke variabel penting seperti total harga. Inheritance diterapkan dengan memisahkan jenis layanan spesifik (seperti Kiloan atau Satuan) dari kelas induk utama. Sementara itu, Polymorphism dimanfaatkan untuk fungsi penghitungan harga yang bisa beradaptasi secara dinamis berdasarkan jenis layanan yang dipanggil.
  • File Persistence: Mengingat aplikasi ini berjalan pada skala operasional tertentu, penyimpanan data tidak menggunakan server SQL eksternal, melainkan memanfaatkan File Persistence (misalnya ObjectOutputStream atau manipulasi teks) untuk memastikan riwayat pesanan tetap utuh dan persisten di dalam sistem komputer lokal meskipun program dihentikan.
Snippet Kode: Polymorphism & Model Dasar
// Contoh kerangka OOP pada Del-Laundry System
public abstract class LayananLaundry implements Serializable {
    private String namaPelanggan;
    private double beratKiloan;

    public LayananLaundry(String nama, double berat) {
        this.namaPelanggan = nama;
        this.beratKiloan = berat;
    }

    // Metode abstrak untuk memicu Polymorphism pada sub-class
    public abstract double hitungTotalHarga();
    
    // Encapsulation
    public String getNamaPelanggan() {
        return namaPelanggan;
    }
}

2. Hotel Reservation System

Sistem reservasi kamar hotel terintegrasi yang mampu mensimulasikan lingkungan transaksi real-time. Tantangan terbesar dalam pengembangan aplikasi sejenis ini adalah memastikan tidak terjadinya tumpang tindih data saat ada pesanan bersamaan.

Sorotan Teknis (Technical Highlights):
  • Multithreading & Concurrency: Di lingkungan perhotelan, puluhan tamu bisa saja mencoba memesan kamar secara bersamaan. Saya mengimplementasikan konsep thread pada Java agar sistem dapat memproses request secara paralel (konkuren) tanpa menyebabkan aplikasi mengalami freeze (macet).
  • Synchronized Methods (Mencegah Race Condition): Untuk mengatasi anomali di mana dua tamu mem-booking kamar yang sama di detik yang sama, saya mengunci metode transaksional menggunakan keyword synchronized. Ini memastikan akses ke database dieksekusi satu per satu pada objek data kamar.
  • PostgreSQL Database & JDBC: Status kamar, riwayat tamu, dan data transaksi diintegrasikan langsung ke dalam DBMS PostgreSQL. Saya menggunakan Java Database Connectivity (JDBC) untuk mengirimkan query, memastikan data selalu sinkron antara kode aplikasi dan database.
Snippet Kode: Pengamanan Thread & Sinkronisasi Data
// Metode synchronized mencegah Race Condition dari akses Multithreading
public synchronized boolean bookKamar(int idKamar, String namaTamu) {
    if (dbManager.cekKetersediaanKamar(idKamar)) {
        
        // Memperbarui database melalui JDBC Connection
        dbManager.updateStatusKamar(idKamar, "BOOKED");
        dbManager.catatReservasiTamu(idKamar, namaTamu);
        
        System.out.println("Kamar " + idKamar + " berhasil diamankan oleh " + namaTamu);
        return true;
        
    } else {
        System.out.println("Gagal: Kamar " + idKamar + " sudah terisi saat ini.");
        return false;
    }
}
